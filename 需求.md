# Markdown 编辑样式迁移需求（对标 IntelliJ Markdown 插件）

## 1. 背景与目标

- 目标分支：`codex/markdown-style-migration`
- 目标：将 JetBrains IntelliJ Markdown 插件的“编辑器语法高亮风格”迁移到当前项目编辑区（非预览区），并结合当前截图进行视觉校准。
- 结果：形成一套可配置、可扩展的 Markdown 样式 token 体系，支持后续持续微调。

- 当前样式实现位置：
  - `Sources/MarkdownEditor/MarkdownSyntaxHighlighter.swift`
  - `Sources/MarkdownEditor/EditorStyleSettings.swift`
  - `Sources/MarkdownEditor/MarkdownTextEditor.swift`
- 当前问题（与目标相比）：
  - 样式语义覆盖不完整，尚未形成与 IntelliJ 类似的“细粒度 token”。
  - 样式参数分散在高亮逻辑中，不利于后续按主题/截图做集中调色。
  - 字号、行高、代码块细节可调维度不够明确。

## 4. 迁移范围（第一阶段）

- 仅处理编辑区（`NSTextView`）语法高亮与文本样式。
- 不改动 Markdown 语法解析能力边界（先保持现有正则方案）。
- 不改动预览区 CSS（`default.css`）作为本阶段主目标。

## 5. 样式 Token 设计（对齐 IntelliJ 语义）

建议将样式拆分为“语义 token -> 具体颜色/字体”的两层：

1. 结构标记类：
- `headerMarker`（`#`）
- `listMarker`（`-` `*` `1.`）
- `blockquoteMarker`（`>`）
- `hrule`
- `tableSeparator`（第二阶段）

2. 文本内容类：
- `headerLevel1...headerLevel6`
- `boldText` / `italicText` / `boldItalicText`
- `strikethroughText`
- `blockquoteText`

3. 链接类：
- `autoLink`
- `linkText`
- `linkDestination`
- `linkLabel`
- `linkTitle`

4. 代码类：
- `inlineCodeText` / `inlineCodeMarker` / `inlineCodeBackground`
- `codeFenceText` / `codeFenceMarker` / `codeFenceLanguage` / `codeFenceBackground`

5. 其他：
- `commentLike`（用于注释风格文本，第二阶段）
- `frontMatterDelimiter`（第二阶段）

## 6. 字体与字号策略（拟定）

- 正文基准字号：沿用编辑器设置 `editorFontSize`（当前默认 14）。
- 标题级别：先不改变字号，仅做颜色+字重差异（与 IntelliJ 编辑器行为一致）。
- 代码字体：保留 `JetBrains Mono` 优先回退链。
- 行高：通过段落样式做轻量优化（保证中文阅读性，不破坏输入体验）。

## 7. 颜色策略

- 采用“浅色/深色双方案”结构，按系统外观切换。
- 第一轮使用对标 IntelliJ 的语义风格，不追求 1:1 色值；
- 第二轮根据截图进行微调，重点校准：
  - 标题文本紫色调
  - 链接蓝色调
  - 代码默认字符串色系（遵循 IntelliJ 语义）
  - 普通正文与弱化文本灰度层级

## 9. 验收标准

- 在同一份 Markdown 样例中，以下元素均有清晰差异化样式：
  - 6 级标题、链接（文本/地址）、行内代码、代码块、引用、列表、删除线、水平线。
- 浅色/深色模式切换后可读性稳定，不出现对比度明显不足。
- 编辑输入、撤销、光标行为不受负面影响（无明显卡顿/闪烁）。
- 字体大小调整后高亮样式同步正确。

## 10. 已确认决策

1. 标题保持 IntelliJ 编辑语义：不放大字号，仅做颜色与字重变化。
2. 代码块背景遵循 IntelliJ 默认语义：取消浅绿色背景，采用代码字符串色系。
3. 链接按语义细分：区分 `linkText` 与 `linkDestination`，并细分 `linkLabel` / `linkTitle`。
4. 第一阶段暂不做表格分隔符与 FrontMatter 的精细上色，放到后续迭代（已在第 11 节完成后续落地）。

## 11. 后续迭代落地（本轮新增，已完成）

1. 表格分隔符与 FrontMatter 精细上色
- 新增表格分隔符高亮：分隔线与 `|` 管道符按独立 token 着色。

- 链接文本支持下划线显示（便于识别可点击区域）。
- 支持 `Command + 鼠标点击` 打开链接；普通点击不抢占编辑光标行为。

3. 输入体验修复（光标跳转）
- 修复中文输入法组合输入期间的光标跳动问题：
  - 组合输入（marked text）期间跳过高亮重绘与外部文本同步。
  - 结束编辑后再统一同步与高亮，保证输入稳定。
  - 移除重复高亮触发路径，避免每次输入多次重绘。
