# AGENTS

## 分支创建以及代码提交规则

1. 创建分支需要加上 `claude/` 前缀，分支名需要简要描述当前分支目的，格式为 `claude/[简短描述，单词以-分割]`。例如：`claude/perf-system-optimization`。
2. 提交信息避免过于冗长的描述或不必要的细节，且务必使用中文。
3. 提交信息应包含 `类型`、`作用域`、`主题`、`正文` 4 个部分。

### 提交信息结构

- 类型：说明本次提交属于哪类改动
  - `feat`：新功能
  - `fix`：修复 Bug
  - `refactor`：重构，不新增功能，也不修复 Bug
  - `docs`：修改文档，比如 README
  - `style`：代码格式变动，不影响功能
  - `test`：添加测试、改测试
  - `chore`：杂项，比如改 `.gitignore`、构建脚本
  - `perf`：性能优化
  - `ci`：CI/CD 相关改动
  - `build`：改构建系统或依赖
  - `revert`：回滚某个提交
- 作用域：说明本次提交修改的模块。
- 主题：总结本次提交内容。
- 正文：说明具体的修改细节。

### 推荐格式

```text
<类型>(<作用域>): <主题>

<正文>
```
